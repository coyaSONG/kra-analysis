<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime probe -> Redis dependency | Optional infrastructure dependency can fail, be missing, or be miswired at runtime. |
| router response -> operator tooling | Operators trust `/health/detailed` for incident diagnosis and must not receive misleading healthy states. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | `apps/api/routers/health.py` Redis probe | mitigate | Reject fake Redis objects without `ping()` and classify failures explicitly as `unavailable` or `error` per D-03. |
| T-01-02 | D | `/health/detailed` route | mitigate | Keep the route fail-open for Redis exceptions and optional dependency absence per D-01 so operator probes do not turn a partial outage into a total blind spot. |
| T-01-03 | R | `apps/api/bootstrap/runtime.py` snapshot output | mitigate | Encode component state strings and overall degraded logic in one facade so tests and runtime share the same truth. |
</threat_model>
