<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client request -> structured logs | Untrusted headers, query params, and request bodies cross into observability output. |
| request logging -> accounting middleware | Request identity crosses middleware boundaries and must stay coherent for audit trails. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | I | `apps/api/middleware/logging.py` log payloads | mitigate | Reuse one shared masking helper for headers, query params, and request bodies; keep body logging to small JSON payloads at debug level only per D-06. |
| T-01-05 | R | request lifecycle events | mitigate | Emit start/complete/fail events from one middleware path with a single request id source per D-04 and D-05. |
| T-01-06 | T | `apps/api/middleware/policy_accounting.py` request id handling | mitigate | Remove competing request-id generation so downstream accounting cannot rewrite or desynchronize the canonical request id. |
</threat_model>
